How the Tree Works

The tree is a map of scene states. Each state represents a moment around the campfire. The player burns items, and the system checks the current state's burns dictionary for a matching entry. If found, dialogue plays and the tree may transition to a new state.

Characters don't spawn automatically — the Attraction System rolls probability each interval. Only when all Required Characters are seated does the current state become active and burns start triggering dialogue.

States are keyed by their State ID in the exported GDScript. Renaming a node after connecting it won't break connections in the editor, but you must keep IDs stable once they're referenced in code.

Node Fields

State IDAuto-generated unique key (e.g. node_3). Used as the dictionary key in the exported GDScript. Cannot be edited — rename the node using Title instead.
TitleHuman-readable label shown on the node card and in connection destinations. Does not affect runtime logic.
DescriptionFlavour text describing what this scene moment is. Written for your own reference — not exported to the game.
Type: SceneA normal state with burn outcomes and dialogue. The game can transition into and out of it.
Type: TerminalAn end state. No burns, no outgoing connections. Characters are dismissed when the tree reaches a terminal node.

Characters

Required CharactersAll of these character IDs must be seated at the fire for this state's burns to trigger dialogue. If any are absent, burns still fire the Attraction System effects but produce no story response.
Eligible PoolCharacters that the Attraction System is allowed to spawn during this state. Only characters in this list will roll their probability. Characters not in the list are ignored entirely, even if their probability is high.
A character can be in the Eligible Pool without being Required. This lets optional characters drift in and add ambient flavour without blocking progression.

Burn Outcomes

Each burn outcome fires when the player burns a matching item at the matching level. The key format is item_id__lv0 (exact level) or item_id__lv* (any level). Exact matches take priority over wildcards.

item_idMust exactly match the burnable_id exported on the Burnable node in Godot.
LevelWhich burn level triggers this outcome. 0 = first burn, 1 = second, etc. Enter * to match any level not covered by an exact entry.
Requires FlagIf set, this outcome only triggers its Dialogue if the named flag is true in the progression state. Otherwise the Blocked Dialogue plays instead. Leave empty for no gating.
DialogueLines spoken when the burn is recognised and the flag requirement is met (or absent). Played in order, one line per advance.
Blocked DialogueLines spoken when the flag requirement is not met. The burn is still recognised — it just plays this alternative response and does not apply Effects or trigger a transition.
EffectsKey/value flags set on the progression state when this burn resolves. Use true or false. These flags can gate future burns via Requires Flag.
Connection (→)Drag the ● port on a burn slot to another node's input port to set the next state. After dialogue finishes, the tree transitions here.
Burns without a connection don't transition — the state stays the same. Useful for ambient reactions that don't advance the story.

Dialogue Lines

Character fieldThe character_id of the speaker. Must match the ID registered via register_character() in Godot. Autocompletes from the node's Eligible Pool.
Text fieldWhat the character says. Supports Godot BBCode tags for animated text effects (see below).
Lines play in order. The player must press the advance input to move to the next line. A state transition only fires after the last line is dismissed.

Fallback (Unrecognised Burn)

Plays when the player burns something that has no matching entry in this state's burns dictionary. Use this to give ambient responses to items you haven't scripted yet, or to give characters a generic reaction that fits the scene's mood.

The Attraction System still receives the item_burned_unrecognized signal, so unrecognised burns still shift character probabilities based on item affinity.

BBCode Animation Tags

Click a tag button above a dialogue line to insert it. Select text first to wrap it, or click with no selection to insert a placeholder. All tags must be closed with the matching [/tag].

💓 pulse
[pulse freq=1.0 color=#ffffff40 ease=-2.0]…[/pulse]
Pulsing opacity/colour. freq: speed (higher = faster). color: target colour multiplier. ease: easing exponent, negative = in-out.
〜 wave
[wave amp=50.0 freq=5.0 connected=1]…[/wave]
Text bobs up and down. amp: height of motion. freq: speed (0 or negative = no wave). connected: 1 = ligatures move together.
🌀 tornado
[tornado radius=10.0 freq=1.0 connected=1]…[/tornado]
Text orbits in a circle. radius: orbit size. freq: speed (0 = paused, negative = reversed). connected: 1 = ligatures move together.
⚡ shake
[shake rate=20.0 level=5 connected=1]…[/shake]
Text shakes randomly. rate: shake speed. level: max offset distance. connected: 1 = ligatures move together.
🌫 fade
[fade start=4 length=14]…[/fade]
Static opacity fade. start: character offset where fade begins. length: how many characters the fade covers.
🌈 rainbow
[rainbow freq=1.0 sat=0.8 val=0.8 speed=1.0]…[/rainbow]
Cycling rainbow colour. freq: letters per repeat. sat/val: saturation and brightness. speed: cycles/sec (0 = paused, negative = reversed).

Saving & Exporting

💾 SaveDownloads campfire_tree.json — a full editor save including node positions, pan/zoom, and all data. Use this to continue editing later.
📂 ImportOpens a save file previously created by Save. Also accepts the older Export JSON format.
Export GDScriptGenerates progression_data.gd — the file your Godot project loads via const DATA = preload(…). Copy this into your project whenever you update the tree.
